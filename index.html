Product / Data Analyst — pre-launch experiment design

AB Test Research Designer

Experiments deserve engineering. Не sample-size calculator, а research-design слой до платформы экспериментов: peeking, SRM, multi-arm risk, CUPED и guardrails ловятся до того, как команда нажмёт Launch.

Live demo ↗ GitHub ↗ Evidence ↗
57%FALSE-DECLARATION rate under peeking (literature) · Johari et al. KDD 2017

57% — это литература, не self-measurement. KDD 2017 «Peeking at A/B Tests» (Johari, Pekelis, Walsh, Koomen) показал: при continuous peeking fixed-horizon тесты декларируют «победу» в большинстве A/A-симуляций. Это причина, по которой нужен research-design слой до эксперимент-платформы — не то, что меряет этот продукт о себе.

Реальный product output (live HF Space, captured Playwright 2026-05-10): baseline 4.2% + MDE 7% → 75 514 / variant, 21 days; group-sequential O'Brien-Fleming, 4 looks, z = 4.05 / 2.86 / 2.34 / 2.02; SRM на 49.0/51.0 split → χ² = 20, p ≈ 8·10⁻⁶; CUPED ρ²=0.25 даёт 591 → 443; Bayesian credible interval — 277/variant.

AI Analyst — production RAG-агент с retrieval и confidence gates

RAG Support Assistant

Без ссылки это не ответ. Вопрос проходит classify → hybrid retrieval → rerank → generate → verify; финальный ответ несёт inline-citations [N], открывает source-panel и route'ится к человеку, если confidence не выдержал.

Live docs ↗ GitHub ↗ Benchmark ↗
0.99CITATION_ACCURACY · 29-case offline grounded oracle

29 ручных кейсов в 10 категориях: warranty, returns, error_codes, installation, billing, general, multi_hop, contradiction_trap, off_topic, insufficient_data. Каждый кейс несёт expected source IDs, expected route и оракульный ответ с явными [N]-цитатами.

Цитаты проверены против retrieved-docs по keyword-stem. Escalation distinguished от рекомендации «обратитесь в поддержку»: триггерят только сильные маркеры — «ответить нельзя», «противоречие», «недостаточно данных». Route distribution: auto 21 / human 8 / retry 0 / error 0.

04 / 04
AI Analyst / LLM Platform — multi-model consensus

GraceKelly

Одна модель может ошибаться уверенно. Один prompt уходит в несколько моделей, ответы сравниваются как claims, embedding-кластеризация показывает agreement, disagreement и instability. Публичная история — Consensus, не каталог режимов.

Live docs ↗ GitHub ↗ Benchmark ↗
1.00UNSTABLE-CLAIM DETECTION · 35-case claim-set simulation

35 кейсов в 8 категориях (factual unanimous / factual strong / ambiguous split / contradictory / trap false consensus / trap majority wrong / insufficient data / unstable). На каждый кейс — 5 симулированных ответов с известными group-labels.

На симулированной similarity-матрице (same-group 0.93, different 0.35, abstain-vs-other 0.20) гоняются живые модули gracekelly.core: clustering_hac, cluster_confidence, divergence. Каждая из 8 категорий получает ровно ту дискриминацию, которую заявляет divergence-порог.

Live LLM не вызывается, ответы шаблонные, similarity синтезирована из group-меток. Доказательство дискриминации consensus-механики, не live-снижения галлюцинаций. false_consensus_rate 0.50 — это известный пол unanimity-based consensus: если все 5 моделей единодушно неправы, движок не имеет ground-truth-оракула. Шкала доказательств

Не все числа равны. Каждое значение на этой странице помечено тем, как оно получено.

live · product

Реальный артефакт из работающего продукта — тесты, бенчмарки на свежих данных, метрики прямо из кода.

offline · oracle

Ручной набор кейсов с эталонными ответами и явными цитатами. Проверяет discipline пайплайна, не live-качество модели.

simulation

Синтетические входы поверх живых core-модулей. Доказывает механику, не реальный продакшен-эффект.

not measured

Прямо отсутствует. Метрики, которые ещё не пройдены через benchmark, на этой странице не появляются.
